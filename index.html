纠结的建小站过程

想要拥有一个可以独立编码的自己的小站是我蓄谋已久的一个想法了，从自己真正接触到HTML CSS和JAVASCRIPT之后，就觉得其实web真的是作为应用的终端的最佳选择，无需虚拟机，跨平台，多语言融合，协议超级通用等等数不清的优势。尤其有了js之后更加生猛无比，非任何终端可以匹敌。windows桌面不行，java平台也不行。

寻找server

在信息时代比较庆幸的一点就是很少走弯路，只要合理使用搜索引擎，就可以得到正确的答案。自然，我一baigoogledu，就将目标瞄准向github这个geek各节的NBserver！多么幸运而幸福的时代~作为geek的平台，自不能没了geek的command&tools横行的恶习。多亏他家的入门文档不错，自心血来潮起直至当夜凌晨，已经可以熟练git shell并且配置成功了自己的git page！那是极其喜悦的时刻~

make static web page work

最直接的就是HTML+JS走起，但通过github看到它家比较推荐jeklly，通过markdown的极易规范语言就可以实现排版编辑等日常博客事物，可无奈windows这破系统在opensource世界里太不招人待见，装几个插件，搭个运行环境都困难重重，及至最后无奈放弃，感叹实非人力所能为，遂放弃此法。但依然寻了些markdown的语言规法来看，发现其实和HTML很像，做了更多的简化而已。但自己转念一想，其实HTML若只是写写博文，po点图，用个浅度定制的框架也是极简单的事情呀~于是便有了现在这个样子的小博客。觉得若要博客长得好看，最主要的还是要内容多多，布局构图配色等倒都只是其次了。希望我的博客能够迅速充实起来
